# Interactive Playground

Enable users to interact with the `POST /notarization` endpoint and preview its functionality.

---

## POST /notarization

Creates a new notarization case for a document.

### Try it out

<div className="playground">
  <div className="request">
    <div>
      <strong>Authorization</strong>
      <input type="text" placeholder="Bearer <token>" className="input"/>
    </div>
    <div>
      <strong>Request Body</strong>
      <textarea className="textarea" placeholder='{
        "jurisdiction": "US",
        "state": "CA",
        "documents": [
          {
            "document_type": "Power of Attorney",
            "document_file": {
              "type": "url",
              "url": "https://example.com/document.pdf"
            },
            "is_already_signed": false,
            "signers": [
              {
                "email": "signer@example.com",

</div>

## Example Request

### cURL Request
```bash
curl --request POST \
  --url https://api.platus.com/notarization \
  --header 'Authorization: Bearer <token>' \
  --header 'Content-Type: application/json' \
  --data '{
    "jurisdiction": "US",
    "state": "CA",
    "documents": [
      {
        "document_type": "Power of Attorney",
        "document_file": {
          "type": "url",
          "url": "https://example.com/document.pdf"
        },
        "is_already_signed": false,
        "signers": [
          {
            "email": "signer@example.com",
            "first_name": "John",
            "last_name": "Doe"
          }
        ]
      }
    ]
  }'
